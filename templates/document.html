Model Documentation Summary

Overview

This model is designed to predict customer churn for an e-commerce platform using a dataset that includes various customer-related features. The model is implemented using a decision tree classifier within a machine learning pipeline that includes preprocessing steps.

Features and Data Preprocessing
  • Data Source: The model uses data from an Excel file (E Commerce Dataset.xlsx), specifically from the second sheet.
  • Missing Values:
    • Numeric features such as Tenure, WarehouseToHome, HourSpendOnApp, CouponUsed, OrderCount, and CashbackAmount have missing values filled with their respective column means.
    • Additional columns, OrderAmountHikeFromlastYear and DaySinceLastOrder, are similarly handled.
  • Data Cleaning:
    • Some categorical values have been replaced to ensure consistency:
      • MaritalStatus: 'Divorced' is replaced with 'Single'.
      • PreferredLoginDevice: 'Phone' and 'Mobile Phone' are both replaced with 'Handphone'.
      • PreferredPaymentMode: 'CC' and 'COD' are replaced with 'Credit Card' and 'Cash on Delivery' respectively.
      • PreferedOrderCat: 'Mobile Phone' is replaced with 'Mobile'.
    • The CustomerID column is removed as it is not used for prediction.
Model Pipeline
  • Preprocessing Steps:
    • One-Hot Encoding: Categorical features are encoded to numerical values using one-hot encoding. This transformation is applied to specific columns based on their indices.
    • Standard Scaling: Numeric features are scaled to standardize their ranges.
  • Classifier: A Decision Tree Classifier is used as the model for making predictions.
Model Training and Evaluation
  • Training: The model is trained on a training dataset (x_train, y_train).
  • Evaluation:
    • Accuracy Score: The model's accuracy is evaluated on a test dataset (x_test).
    • Cross-Validation: The model's performance is validated using cross-validation with 5 folds to ensure robustness and generalization.
  • Hyperparameter Tuning:
    • Grid Search is used to find the best hyperparameters for the Decision Tree Classifier.
    • The best parameters and model performance are reported.
Model Export

The trained model pipeline is saved to a file (churn_prediction_model.pkl) using pickle for future use and deployment.

Real-World Applications
  • Customer Retention: E-commerce Platforms: This model can be used by online retailers to predict which customers are likely to stop using their services. By identifying these customers, businesses can implement targeted retention strategies to improve customer loyalty and reduce churn rates.
  • Marketing Strategies: Personalized Marketing: Companies can use the predictions to tailor marketing campaigns. For instance, special offers or incentives can be given to customers predicted to churn in an effort to retain them.
  • Resource Allocation: Customer Support: Resources can be allocated more effectively by focusing on high-risk customers who are predicted to churn. This helps in prioritizing support and intervention efforts.
  • Strategic Planning: Business Strategy: Insights from the model can inform strategic decisions regarding customer service improvements, product offerings, and overall business strategy.
  • Subscription Services: Media and Entertainment: Streaming services or subscription-based platforms can use this model to forecast which subscribers may cancel their subscriptions, allowing them to proactively address potential issues and improve customer satisfaction.

By leveraging this model, businesses can gain valuable insights into customer behavior, enabling them to take proactive measures to enhance customer retention and drive long-term growth.
